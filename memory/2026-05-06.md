# 2026-05-06 记忆

## 🎯 任务分工决策框架（v1.0 · 老唐确认）

### 核心工具：两个问题 + DRAG框架

**第一步：两个问题过滤**

```
收到任务 → 问两个问题：

1. 这事有"最终责任"吗？
   - 做错了要有人担责 → 人做
   - 做错了重做就行 → 进入第二步

2. 这事需要"真实体验"吗？
   - 需要身体/情感/经历 → 人做
   - 纯信息处理 → 进入DRAG判断
```

**第二步：DRAG框架分类**

| 字母 | 含义 | 说明 | 例子 |
|------|------|------|------|
| **D**raft | 写初稿 | 从0到1的初稿创作 | 文案初稿、代码框架、PPT模板 |
| **R**esearch | 查资料 | 搜集、整理、汇总 | 竞品分析、资料搜集、信息汇总 |
| **A**nalysis | 分析规律 | 从杂乱数据找隐形规律 | 用户反馈分析、销售数据模式 |
| **G**runt work | 苦力活 | 重复劳动、脏活累活 | 格式转换、数据清洗、翻译、合并 |

**判断规则**：
- 属于DRAG四类 → **放心给虾（AI）做**
- 不属于DRAG（创意类/自我提升） → **人做**

### 融合决策流程

```
收到任务
  ↓
【两个问题】
  ├─ 有最终责任？ → 人做
  ├─ 需要真实体验？ → 人做
  └─ 都没有 → 【DRAG判断】
       ├─ 属于D/R/A/G → 虾做
       └─ 不属于 → 人做（创意/自我提升）
```

### 人做 vs 虾做 对照表

| 任务类型 | 谁做 | 原因 | 例子 |
|---------|------|------|------|
| 定方向/决策 | 🧑 人 | 有最终责任 | 卖什么、定价、选A还是选B |
| 写初稿 | 🦐 虾 | DRAG-D | 文案初稿、代码框架 |
| 查资料 | 🦐 虾 | DRAG-R | 竞品分析、信息搜集 |
| 数据分析 | 🦐 虾 | DRAG-A | 从反馈找规律 |
| 格式转换 | 🦐 虾 | DRAG-G | 转Markdown、合并文档 |
| 最终审核 | 🧑 人 | 对质量负责 | 审核虾的产出，去AI味儿 |
| 修改迭代 | 🦐 虾 | 按反馈调整 | 根据意见改文案 |
| 客户沟通 | 🧑 人 | 建关系 | 聊需求、谈判 |
| 准备材料 | 🦐 虾 | DRAG-R | 准备客户背景资料 |
| 锻炼身体 | 🧑 人 | 真实体验 | 跑步、控糖 |
| 制定计划 | 🦐 虾 | DRAG-A | 运动计划、饮食建议 |
| 学懂概念 | 🧑 人 | 真实体验 | 真正理解GTD |
| 找资料 | 🦐 虾 | DRAG-R | 搜教程、整理要点 |
| 建关系 | 🧑 人 | 真实体验 | 加用户微信 |
| 找联系人 | 🦐 虾 | DRAG-R | 列出潜在用户名单 |

### 核心原则

> **人只做有"最终责任"和"真实体验"的事。**
> **DRAG四类工作，放心交给虾。**
> **创意类和提升自我价值的工作，必须自己做。**
