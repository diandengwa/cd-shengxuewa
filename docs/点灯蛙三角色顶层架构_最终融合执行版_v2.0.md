## 核心战略定论

**点灯蛙不是"三产品矩阵"，而是"一个收费主产品 + 两个免费流量资产 + 一个社区飞轮"。**

> 升学参谋是唯一收钱的产品，AI思维教练是喂进AI和搜索引擎的免费获客发动机，SchoolZone是独立子域名的SEO磁铁，养虾社区是把付费用户变成传播者的私域内核。去AI化是品牌+合规双保险，生成式AI备案是收款前必须扫清的硬门槛，拟人化互动办法是设计红线的法律锚点。一人公司的全部胜算，在于只运营一条变现线，把另外两个角色变成不花钱却持续供血的上游和下游，同时把合规前置做到"先于监管、不踩红线"。

## 三、三角色最终处理方案

### 角色①：升学参谋——唯一变现，去AI化到骨子里

**定论**：www.diandengwa.com 唯一商业产品，所有收入从这一条线产生。

**产品形态**：
- 着陆页：单产品聚焦，3秒传达"成都升学，到底行不行？让参谋帮你四步裁决"
- 工作台SPA：诊断Tab（四步裁决+8段报告+付费墙）+ 政策Tab（日历/黑话/划片跳转SchoolZone）+ 我的Tab
- 支付页：PRO ¥99/月或¥999/年，¥9.9首月体验仅邀请码限量发放

**去AI化双重价值**：
- 品牌价值：严肃决策场景，专家定位增强信任
- 合规价值：在产品定位文本层面提前排除适用《拟人化互动办法》的可能性

**对外口径**（钉死）：本服务为基于公开政策数据与规则辅助分析的升学研判工具，非持续性情感互动服务。

### 角色②：AI思维教练——拆两层，飞轮运转

**定论**：不做独立产品，不完全折叠为Pro权益，拆成两层。

**公开层（免费·可传播）**：
- 载体：github.com/diandengwa/diandengwa + 公众号
- 内容：四学段提示词、灵魂追问模板、家长案例、豆包/DeepSeek/千问配置指南
- KPI：被AI问答引用、被家长转发、Stars、Discussions活跃度
- 命名：叫"AI思维教练配置包"安全，叫"AI陪伴学伴"要谨慎（避免滑向拟人化定义）

**付费层（Pro个性化交付）**：
- 作为Pro诊断报告第8段增值模块，基于用户画像做个性化组合
- 不是复制开源提示词，而是"你家孩子处于小升初衔接阶段，建议优先训练题意复述、错因归类，以下是适合本阶段的3个追问模板"
- 产品里标注"本建议综合了点灯蛙共创社区中XX位家长的实践反馈"，用署名致敬替代"被白嫖感"

**飞轮逻辑**：社区贡献 → 开源库沉淀 → 主站Pro个性化调用 → 付费用户回社区贡献案例

### 角色③：养虾教练——彻底退为品牌哲学

**定论**：不做产品功能，不做独立入口，不进入工作台。唯一落地处是 /about 品牌故事页。

**三重理由叠加**：
1. 资源约束：一人公司无法支撑第三产品线
2. 市场验证：养虾群40人已近沉默，独立运营不可行
3. 合规风险：VIS v4.0的IP成长阶段矩阵（Lv1萌动期→Lv4智睿期+智慧之光成长隐喻）若做成产品内"养一只属于自己的蛙"长期陪伴养成机制，正中《拟人化互动办法》靶心。**退为品牌叙事不是权宜之计，是合规既定方针，Phase 2/3不可重新讨论"要不要做成陪伴养成"**

**保留价值**：养虾作为社群文化暗号在共创社区内延续；公众号"老万点灯"养虾日记作为人设内容持续输出；/about静态叙事。

## 四、合规体系：三道红线

### 4.1 第一顺位（P0·收钱前硬门槛）：生成式AI服务备案

**事实**：升学参谋使用大模型对公众生成诊断内容，属于面向公众的生成式人工智能服务。"清朗·整治AI应用乱象"专项行动第一阶段第一条整治"应备未备"，已有应用被约谈并行政处罚。

**行动**：在收第一笔钱之前，必须确认备案/登记义务口径。这不是锦上添花，是P0阻塞项。

**三步自查框架（找律师之前自己先判断清楚）**：

**第一步：是否只调用已备案模型？** 是（如DeepSeek/通义/文心等已完成备案的模型）→ 不需要自己做大模型备案，但**仍需在产品页面显著位置公示所调用模型的名称和备案号/登记编号**。这个动作今天就能做，零成本，直接写在页脚或 `/compliance` 页面。

**第二步：服务是否具有舆论属性或社会动员能力？** 升学参谋面向单个家庭，输出个人化路径建议，不是公共政策评论、不是大规模内容分发 → 大概率不具有。但这个判断要明确（写一份内部评估备查，或让律师出意见信），不是"我觉得没问题"。

**第三步：属地网信（四川省网信办）口径？** 调用已备案模型的应用，是无需登记、应用/功能登记，还是算法备案？这一步自己判断不了，**必须问人**——直接问属地网信（可以"咨询"名义问），或找做过类似案例的合规律师/代办机构问，成本几百到几千元，值得花。

**备案未确认前的底线**：
- 使用已备案、合规可商用的国内模型
- **在合规页面公示所调用模型名称+备案号（今天就能做，零成本）**
- 保留内容安全审核、敏感词拦截、人工复核机制说明
- 不做开放式公共发布社区，不让用户一键发布生成内容到公网
- 不做新闻/公共政策舆论评论型输出，只做家庭升学参考建议
- **可小范围灰度收款验证PMF，但不做任何公开投放和规模化拉新**

### 4.2 第二顺位（P0.5·8月底前必须有答案）：ICP许可证

**事实**：升学参谋的付费解锁报告可能构成经营性互联网信息服务，需判断是否需要ICP增值电信业务许可证（信息服务业务类）。这不是"以后要不要做"的问题，是"现在敢不敢收第一笔钱"的问题。

**行动**：8月底前必须拿到一个专业意见。不阻塞上线，但要**带着一个明确的法律意见**做上线决策，而不是"不知道有这件事地"上线。给自己一个明确的截止日期，不要无限期搁置成"以后再说"。

### 4.3 第三顺位（产品设计红线）：拟人化互动规避

**事实**：《人工智能拟人化互动服务管理暂行办法》2026年7月15日施行，明确豁免"学习教育服务"，但前提是"不涉及持续性情感互动"。

| 产品线 | 是否落入 | 设计要求 |
|--------|----------|----------|
| 升学参谋 | 不落入 | 本质是单次/低频决策诊断工具，定位为"参谋"而非"伙伴" |
| AI思维教练 | 不落入 | 核心是"引导思考的工具配置包"。**红线：可做任务状态反馈，不能做关系养成** |
| 养虾 | 不触及 | 已退为品牌叙事，无产品功能 |

**设计红线**：可以有表情反馈（专注/灵感），不能有关系养成（亲密度/等级/陪伴天数/关系升级）

### 4.4 第四顺位（运营红线）：GEO白帽

**事实**：清朗行动明确整治"使用GEO技术恶意营销、实施AI数据投毒"

| 允许 | 禁止 |
|------|------|
| 原创政策解释 | 批量灌关键词 |
| 引用官方来源 | 伪造权威来源 |
| 标注链接、交叉验证 | 制造虚假引用 |
| 真实案例匿名化 | 内容农场 |
| 结构化FAQ | 让AI大量生成低质文章 |
| 引用信源标注 | 不标注引用来源 |

### 4.4 未成年人保护（前置）

即使服务对象是家长，内容涉及未成年人教育，仍需保护机制：
- 默认不采集孩子姓名、身份证号、学校班级等非必要信息
- 家庭画像字段尽量用枚举，不用自由文本收集隐私
- 报告中不做对孩子人格、智力、心理状态的标签化判断
- 不承诺录取、不制造焦虑、不做学校优劣绝对排名
- 支付页明确由成年人购买
- **用户输入不直接进入训练数据**：隐私政策明确"用户输入用于生成本次报告、保存历史和改善服务质量；如未来用于产品改进，必须做匿名化处理或另行取得授权"

### 6.4 核心产品闭环

```

  → 输出1段核心判断（明确但不完整）
  → 提示：你的情况需要深度研判
  → PRO付费
  → 8段诊断报告
  → 保存历史报告 + 加入共创社区
```

**Free层不能太薄**（用户不付费）也**不能太厚**（无付费理由）。最佳：给出明确但不完整的判断，把路径组合、风险权重、时间线、学校匹配放到Pro。

**Free层输出示例**：
> 你家情况的核心变量不是户籍，而是居住证登记地址与入学区域之间的匹配关系。基于你输入的信息，孩子有资格走随迁入学路径，但具体路径选择、风险点、时间线需要更深入的分析。

这段话**明确**（告诉用户关键变量是什么）但**不完整**（没给路径组合、风险权重、时间线），给用户付费理由但不制造焦虑。

## 十、IP形象使用边界

四态表情（专注/灵感/疲惫/庆祝）**视觉上可备齐**，但使用上分阶段且严格区分"任务状态"与"关系状态"：

| 阶段 | 使用范围 | 禁止 |
|------|----------|------|
| Phase 1 | 专注态（诊断进行中）、疲惫态（出错/超时） | 不做关系叙事、不做成长养成 |
| Phase 2 | 四态完整使用 | 仍不做养成机制，重新核验拟人化执法口径 |
| Phase 3 | 待评估 | 养虾产品化前强制做"拟人化合规自查" |

**核心原则**：表情表达的是**任务状态**（"正在分析""遇到困难"），不是**关系状态**（"我们越来越熟了""蛙蛙想你了"）。

**场景映射表**：

| 场景 | 表情态 | 文案 |
|------|--------|------|
| 报告生成中 | 专注态（铅笔在写） | 参谋正在研判… |
| 加载超时/出错 | 疲惫态 | 系统正忙，请稍候 |
| 报告生成完成 | 专注态（铅笔指向报告） | 研判完成，查看报告 |
| 支付成功 | 暂用系统图标（不用庆祝态） | 已解锁完整研判报告 |

> Phase 1只用专注+疲惫两态。灵感态和庆祝态视觉上可备齐，但Phase 1先不使用——庆祝态在支付成功场景可能产生"陪伴/奖励"暗示，用系统确认图标更安全。
